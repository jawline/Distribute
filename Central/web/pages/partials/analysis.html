Analysis

Many different analysis techniques are used across Cyber Security. The purpose of the analysis varies depending on the area, as do the techniques used to perform the analysis. Two analysis techniques closely related to Cyber Security are listed below.

Network Analysis

Analysing the traffic on a network the size of the internet is an incredibly challenging task. There are many different companies and individuals working to monitor web traffic in order to identify potential threats and provide a 'map' or visualization of some data about the network. One such tool is the live attack map provided by the company NORSE[3], which provides real time information on DDOS attacks which they are tracking throughout the world.

Program Analysis

Undetected vulnerabilities and bugs in software have gone from being a minor annoyance to the root cause of many serious security breaches in the last decade. In order to combat this new threat tools which can automatically detect potential issues with software have become popular, allowing developers to spot issues with their products before they ever enter production.

[2]

Static Analysis

Static analysis tools attempt to reason about how a program will behave by examining the program code.

On simple pieces of code various types of deterministic static analysis can be used to find common programmer errors such as uninitialized variables or unsafe casting. These tools often cannot find complex issues in a codebase however, as many pieces of code cannot be reasoned about deterministically.

For larger pieces of code or complex vulnerabilities heuristic static analysis techniques can be used to find issues. These techniques can find more complex issues in code, however they do not reason about the code perfectly and so they can produce false positives.

Dynamic Analysis

One other popular program analysis technique is dynamic program analysis. In dynamic program analysis the program is executed and the program execution or 'path' is analysed. This analysis technique can expose errors which are commonly missed by other techniques and does not produce false positives, however it is often more computationally expensive then other techniques. It is also unsuited to testing certain types of software, like software that loops indefinitely.
